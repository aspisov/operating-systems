## Условие
### Задача о нелюдимых садовниках. 

Имеется пустой участок земли (двумерный массив размером M × N ) и план сада, разбитого на отдельные квадраты. От 10 до 30 процентов (задается случайно) площади сада заняты прудами или камнями. То есть недоступны для ухаживания. Эти квадраты располагаются на плане произвольным (случайным) образом. Ухаживание за садом выполняют два садовника, которые не хотят встречаться друг другом (то есть, одновременно появляться в одном и том же квадрате). Первый садовник начинает работу с верхнего левого угла сада и перемещается слева направо, сделав ряд, он спускается вниз и идет в обратном направлении, пропуская обработанные участки. Второй садовник начинает работу с нижнего правого угла сада и перемещается снизу вверх, сделав ряд, он перемещается влево и также идет в обратную сторону. Если садовник видит, что участок сада уже обработан другим садовником или является необрабатываемым, он идет дальше. Если по пути какой-то участок занят другим садовником, то садовник ожидает когда участок освободится, чтобы пройти дальше на доступный ему необработанный участок. Садовники должны работать одновременно со скоростями, определяемыми как параметры задачи. Прохождение через любой квадрат занимает некоторое время, которое задается константой, меньшей чем времена обработки и принимается за единицу времени.

Создать многопроцессное приложение, моделирующее работу садовников.

Каждый садовник — это отдельный процесс.

## Описание сценария
В данной задаче моделируется поведение садовников, где каждая сущность представлена отдельным процессом:
- **Садовник 1** движется зигзагообразно по горизонтали и обрабатывает клетки (кроме камней и пруда).
- **Садовник 2* движется зигзагообразно по вертикали и обрабатывает клетки поля (также кроме камней и пруда).
- Садовники не обрабатывают уже обработанные клетки поля.
- Если садовник заходит в клетку с другим садовником, он ждёт, пока эта клетка освободится.
- По завершению обработки всех клеток программа завершается.

## Реализация
### Общие аспекты
- Программа использует множество процессов, каждый из которых выполняет роль садовников.
- Связь между процессами и организация клеток поля осуществляется через семафоры.

### Использование именованных семафоров (4–5 баллов)
### Инициализация семафоров:
- При старте программы для каждой клетки сада создаётся именованный семафор. Это позволяет легко ссылаться на семафоры по уникальному имени, основанному на позиции семафора в массиве (`M x N`).
- Пример имени семафора: `/sem_garden_i_j`, где `i` и `j` — координаты клетки в массиве.
- Семафоры инициализируются в состоянии "открыто", что позволяет первому садовнику войти в соответствующую клетку без задержек.

### Работа с семафорами:
- Перед входом в клетку каждый садовник вызывает операцию `sem_wait` на соответствующем семафоре, что уменьшает значение семафора на единицу. Если значение семафора было равно 0, садовник блокируется до тех пор, пока другой садовник не освободит клетку.
- После завершения обработки клетки садовник вызывает операцию `sem_post`, которая увеличивает значение семафора на единицу, освобождая клетку для доступа другого процесса.

### Завершение работы и освобождение ресурсов:
- По завершении работы всех садовников программа закрывает и удаляет все именованные семафоры используя функции `sem_close` и `sem_unlink`.
- Это предотвращает утечку ресурсов и гарантирует, что все системные ресурсы, связанные с семафорами, корректно освобождаются.

### Использование неименованных семафоров (6–7 баллов)

### Объяснение изменений:

- Использован массив неименованных семафоров `sem_t* semaphores`, который инициализируется и используется в разделяемой памяти.
- Функция `init_semaphores()` инициализирует и возвращает массив семафоров.
- Функция `destroy_semaphores()` уничтожает семафоры после завершения работы программы.
- Функция `gardener_work()` теперь принимает дополнительный параметр `semaphores` для доступа к массиву семафоров.
- Перед доступом к клетке садовником выполняется `sem_wait()` для захвата семафора, и после обработки клетки выполняется `sem_post()` для освобождения семафора.
- Использование `MAP_SHARED | MAP_ANONYMOUS` при вызове `mmap()` для создания разделяемой памяти, доступной всем процессам.
